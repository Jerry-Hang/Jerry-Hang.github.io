这是我的第一篇博客文章，现在左侧目录应该能顺利打开它了。

欢迎来到我的小站！以后的技术分享以及折腾我都放这了

这个是我基于GitHub Pages托管的个人静态博客，具体的代码以及文件我已经在Github上开源了（https://github.com/Jerry-Hang/Jerry-Hang.github.io），需要的话可以自行Fork。接下来我将讲解如何将我开源的个人博客项目制作成你自己的个人博客：
具体的方法有两个；
方案 A：GitHub 直接 Fork（最推荐）
Fork 我的仓库：点击我仓库页面右上角的 Fork 按钮，复制一份到自己账号下。

改仓库名：在 Fork 后的仓库 Settings 里，把仓库名改成 自己的用户名.github.io。

改配置文件：编辑 _config.yml，把 title、author、description 改成自己的。

换头像和背景：在仓库的 assets/images/ 文件夹里，点击 Upload files，上传自己的 avatar.jpg 和 bg.jpg 覆盖原文件。

开启 Pages：在 Settings → Pages 里，选择 main 分支，点 Save。

等待生效：等一两分钟，访问 https://自己的用户名.github.io 就能看到博客。

这一步全程在 GitHub 网页上操作，不需要安装任何软件！

方案 B：用 GitHub Desktop（适合想本地写文章的人）
Clone 我的仓库：在 GitHub Desktop 里选择 Clone repository，输入你的仓库地址。

改配置、换图片：在本地文件夹里修改 _config.yml 和替换 assets/images/ 下的图片。

Push 到自己的仓库：在 GitHub Desktop 里修改 Repository → Repository settings，把远程地址改成自己 Fork 后的仓库地址，然后 Push。

开启 Pages：同上。

如何写文章（日常使用）
在线写：在 GitHub 网页上进入 _posts 文件夹，点击 Add file → Create new file，按格式命名 年-月-日-标题.md，写入 Markdown 内容即可。

本地写：用任何文本编辑器写 .md 文件，保存后用 GitHub Desktop Push。

格式示范：给一个简单的文章模板，包括标题、日期、标签、正文，以及如何插入图片和使用倾斜特效。
